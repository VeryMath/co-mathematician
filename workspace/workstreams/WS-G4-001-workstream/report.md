# 综合分析：PDHG vs Douglas-Rachford 算法选择决策框架

## 1. 核心问题回顾

**研究问题**: 对于给定的连续优化问题，如何判断该使用 PDHG 还是 Douglas-Rachford splitting？
两者互为对偶，数学上等价，但实际效率差异巨大。

**答案**: 选择取决于问题结构、算子性质、计算资源和调参投入。本文综合文献调研 (G1)、
理论分析 (G2) 和计算实验 (G3)，提出系统性的决策框架和量化指标。

## 2. 决策框架

### 第一步：问题形式化

将问题写为标准形式: $\min_x f(Ax) + g(x)$

- $A = I$: 进入 DRS 主路径
- $A \neq I$: 进入 PDHG 主路径，但 DRS 仍可作为备选

### 第二步：检查关键指标

按优先级依次检查：

#### 指标 1: Prox 代价比

$$R = \frac{\text{cost}(\text{prox}_{f^*})}{\text{cost}(\text{prox}_f)}$$

| R 范围 | 推荐 |
|--------|------|
| $R < 0.5$ | **PDHG** — $f^*$ 的 prox 明显更容易 |
| $0.5 \leq R \leq 2$ | 皆可，继续检查后续指标 |
| $R > 2$ | **DRS** — $f$ 的 prox 明显更容易 |

**典型实例**:
- $f = \frac{1}{2}\|Ax-b\|^2$, $R < 0.1$ → PDHG (实验: 11× faster)
- $f = \|\cdot\|_1$, $R \approx 1$ → 均可
- $f = \|D\cdot\|_1$ (TV), $R < 0.05$ → PDHG (实验: 24× faster)

#### 指标 2: 算子范数条件数

$$\kappa = \frac{\|A\|}{\max(1, \sqrt{\mu_g \mu_{f^*}})}$$

| $\kappa$ 范围 | 推荐 |
|---|---|
| $\kappa < 1$ | PDHG — 步长约束宽松，收敛快 |
| $1 \leq \kappa \leq 10$ | 取决于其他指标 |
| $\kappa > 10$ | DRS — 避免 $\tau\sigma < 1/\|A\|^2$ 过度限制步长 |

**重要**: 若 $A$ 非列满秩，DRS 中 $f \circ A$ 失去强凸性，可能丧失线性收敛 — 此时即使 $\kappa$ 大也应优先考虑 PDHG 或预条件方案。

#### 指标 3: 预条件需求

| 场景 | 推荐 |
|------|------|
| 需要预条件 (异构问题) | **PDHG** — 对角线预条件有闭式 |
| 不需要预条件 | 无关 |
| DRS 预条件可行 | 需手动设计，复杂度高 |

**PDHG 对角线预条件** (Pock & Chambolle 2011):
$$\tau_j = \frac{1}{\sum_i |A_{ij}|}, \quad \sigma_i = \frac{1}{\sum_j |A_{ij}|}$$

无需计算 $\|A\|$，对异构尺度问题显著加速。

#### 指标 4: 调参资源

| 场景 | 推荐 |
|------|------|
| 调参时间充足 + 专家知识 | PDHG — 3 参数，调优后性能上限高 |
| 快速原型 / 非专家用户 | DRS — 2 参数，不敏感 |
| 零调参需求 | **免调参 PDHG** (Wang et al. 2024) 或 **松弛 PDHG + 线搜索** (McManus et al. 2025) |

#### 指标 5: 随机扩展需求

| 场景 | 推荐 |
|------|------|
| 大规模数据 → 随机梯度 | SPDHG — 理论更成熟 |
| 逆问题 / 图像重建 | SPDR — 最近 benchmark 优势 (Dong et al. 2024) |

### 第三步：综合决策

```
问题: min f(Ax) + g(x)
│
├─ A = I? ──── 是 ──→ 用指标1判断 R
│                      ├─ R < 0.5: PDHG
│                      └─ R ≥ 0.5: DRS (默认推荐)
│
├─ A ≠ I:
│   ├─ f*(y) 的 prox 有闭式? ── 是 ──→ PDHG (通常是正确选择)
│   ├─ ||A|| 很大 (κ > 10)? ── 是 ──→ 检查 A 是否列满秩
│   │                                    ├─ 是: DRS
│   │                                    └─ 否: PDHG + 预条件
│   ├─ 需要预条件? ── 是 ──→ PDHG + 对角预条件
│   └─ 调参资源有限? ──→ 免调参 PDHG 或 DRS
```

## 3. 评价指标体系

建议使用以下量化指标进行系统性比较：

### 3.1 收敛速度指标

| 指标 | 定义 | 适用场景 |
|------|------|---------|
| 固定点残差 | $\|z^{k+1} - z^k\|$ | 通用，衡量迭代稳定性 |
| Primal-dual gap | $\mathcal{L}(x^k, y^*) - \mathcal{L}(x^*, y^k)$ | 鞍点问题，PDHG 天然支持 |
| 目标函数值差距 | $|F(x^k) - F^*|/|F^*|$ | 已知最优值时 |

### 3.2 计算效率指标

| 指标 | 定义 |
|------|------|
| 每步 wall-clock 时间 | 包含 prox + matvec + 通信 |
| 达到目标精度的总时间 | 综合收敛速度和每步代价 |
| 参数搜索代价 | 找到可行参数所需的试错次数 |

### 3.3 鲁棒性指标

| 指标 | 定义 |
|------|------|
| 参数敏感性 | $\partial(\text{收敛速度}) / \partial(\text{参数})$ |
| 问题缩放稳定性 | 问题规模增加时性能退化率 |
| 噪声容忍度 | 随机梯度/扰动下的性能 |

### 3.4 工程指标

| 指标 | 定义 |
|------|------|
| 实现复杂度 (代码行数) | 开发和维护成本 |
| 依赖复杂度 | 是否需要额外库 (如线性求解器) |
| 内存占用 | 大尺度问题下的内存需求 |

## 4. 实验验证总结

| 问题 | 主导因素 | 推荐算法 | 实验加速比 (PDHG vs DRS) |
|------|---------|---------|--------------------------|
| LASSO | $R \ll 1$ | PDHG | 11× |
| TV 去噪 | $R \ll 1$ + 预条件 | PDHG | 24× |
| 矩阵补全 | $R \approx 1$ | 均可 | 1.1× |

**未来验证方向**:
- 大 $\kappa$ 场景 (PDHG 步长受限) 验证 DRS 优势
- 免调参 PDHG vs 免调参 DRS 对比
- 更大规模问题 (n > 10⁴)

## 5. 方法论建议

### 对于新问题的分析流程

1. 写出 primal 和 dual 形式
2. 计算 $R$ (prox 代价比) 和 $\kappa$ (算子范数条件数)
3. 若 $R < 0.5$: 先尝试 PDHG
4. 若 $\kappa > 10$ 且 $A$ 列满秩: 先尝试 DRS
5. 若不确定: 用 DRS 快速原型 (2 参数)，必要时切换 PDHG 优化
6. 报告时同时汇报 R, κ, wall-clock time 和参数敏感性

### 注意事项

- **不要仅比较迭代次数** — 每次迭代的代价差异可能巨大 (TV 实验中 DRS 单步需 50× 内层迭代)
- **注意 $A$ 的秩** — $A$ 非满秩时 DRS 可能失去理论优越性
- **利用等价性** — 一个算法的参数可通过等价关系转换为另一算法的参数，便于交叉验证
- **考虑工程因素** — 线性系统求解 vs 矩阵-向量乘法在实现上的差异可能超过理论收敛速度差异

## 6. 研究空白与未来方向

1. $\kappa \gg 1$ 场景的系统性实验验证
2. 非凸问题中的等价性是否保持
3. Nesterov 加速在两个框架下的精确对应
4. 面向深度学习优化 (mini-batch, 分布式) 的变体比较
5. 自动算法选择工具的开发

## 7. 结论

PDHG 和 DRS 的"对偶性"意味着理论上没有绝对的优劣，但实际问题中的计算结构
导致清晰的选择偏好：

- **大多数实际场景下 PDHG 更优** — 因为 $f^*$ 的 prox 往往比 $f \circ A$ 的
  prox 更容易计算
- **DRS 在以下情况有优势**: (1) $A = I$ 的简单复合问题, (2) $\|A\|$ 极大且
  $A$ 列满秩, (3) 需要快速原型验证
- **现代方法 (免调参 PDHG) 正在消除 PDHG 的主要劣势 (调参困难)**

最终建议: 从 PDHG 开始，仅在明确满足 DRS 优势条件时切换。
